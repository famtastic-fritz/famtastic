# Learn: Wave 4 Mission Control Observability

- A useful cockpit can start as a deterministic projection object before any UI work. This keeps Desktop/Mission Control future work from inventing storage or coupling directly to filesystem layout.
- Needs-Fritz and stale/blocked can be derived from existing job/witness/ledger semantics without adding new canonical state yet.
- Proof discovery should start with job output artifacts and ledger proof rows; broader proof indexing can be added later once the current projection shape is consumed by UI or briefings.
