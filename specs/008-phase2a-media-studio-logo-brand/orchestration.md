# Phase 2A Orchestration Plan

## Roles

Shay: orchestrator, spec keeper, cost governor, post-eval owner.
Research scout: brand/category/tool/workflow research.
Design scout: visual language, logo critique, UX patterns.
Media scout: MuAPI/Open Generative/Remotion/provider capability mapping.
Writer lane: builds Media Studio workflow screens and backend contracts.
Reviewer lane: verifies SDD compliance, UI usefulness, production-readiness gates.

## Execution model

1. Read existing Phase 2 plan, logo artifacts, and research packet.
2. Create/update SDD docs before implementation.
3. Build backend contracts first only where screens need them.
4. Prototype visible Logo Lab workflow before adding broad Media Studio features.
5. Keep generation dry-run until Fritz approves spend.
6. After each meaningful run, call post-eval and capture opportunities.

## Cost controls

- Use existing logo WIPs first.
- Research/cache before paid generation.
- Dry-run provider queue by default.
- Generate small batches only after Fritz approves direction.
- Preserve all prompts/results so winning patterns are reused.

## Parallelism

Run scouts in parallel, but only one writer lane touches product files at a time. Reviewer lane runs after each implementation slice.
