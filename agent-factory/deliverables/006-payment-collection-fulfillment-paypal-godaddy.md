## Payment rail: PayPal Business
- Receiving account: **[your-paypal-business-email]** (PayPal Business).
- **Primary method — PayPal Invoices** (no code, works today):
  1. PayPal Business → **Invoicing → Create Invoice**.
  2. Bill-to: client email. Line item: "Checkout Rescue Diagnostic — 48h" $500.
  3. Terms: **Due on receipt.** Enable partial-payment OFF. Turn ON the reminder.
  4. Send. Delivery clock starts when status flips to **Paid**.
- **Optional later — PayPal Checkout link / button** for the productized audit
  ($149) so the tripwire is self-serve. Use a **Payment Link** from the PayPal
  dashboard; drop it on the offer page. (Live API keys go in `.env`:
  `PAYPAL_CLIENT_ID`, `PAYPAL_CLIENT_SECRET` — see SETUP.md. The factory never
  calls these; they're for your storefront.)
- **Deposit policy for builds:** 50% to start, 50% on approval. Two invoices,
  same flow. Never start a build before the deposit clears.

## Invoice → cash state machine
`draft → sent → paid → delivered → (care-plan: subscribed)`
- `sent` but unpaid 48h → automatic PayPal reminder + one personal nudge.
- `paid` → send intake form, request access, start the 48h clock.
- `delivered` → send the care-plan opt-out offer + ask for one referral.

## Subscriptions (care plan MRR)
- Use **PayPal Subscriptions** (create a recurring plan in the dashboard) at
  $79/$129/$199/mo. Self-serve link on the post-delivery email.
- Dunning: PayPal retries failed charges automatically; on hard fail, pause the
  care service and email the client.

## Fulfillment pipeline (diagnostic, 48h)
1. **Intake (0–2h):** store URL, admin/staging access, payment processor in use,
   the specific symptom. Standard intake form below.
2. **Audit (2–24h):** walk the full checkout; log every step + drop-off; review
   gateway/plugin config, fraud rules, tokenization/recurring.
3. **Synthesize (24–36h):** prioritized fix plan (impact × effort), the report.
4. **Loom (36–44h):** 3–5 min narrated walkthrough of leak → fix.
5. **Deliver (44–48h):** report PDF + Loom + fix plan; offer the sprint with the
   $500 credit; ask for the referral.

## Fulfillment pipeline (FAMtastic build)
1. Deposit paid → brief intake (brand, pages, vertical, references).
2. **Factory build** via the FAMtastic site studio (the existing pipeline).
3. Human review pass (hero vocabulary, nav skeleton, logo wiring, no inline
   styles — the factory's own non-negotiables).
4. Staging link to client → 1 revision round (Signature+).
5. Final invoice (50%) → on payment, deploy to their domain.
6. Offer care plan → subscribe → recurring revenue.

## Receipts, taxes, records
- PayPal generates the receipt automatically; keep the PDF.
- Log each sale in the simple CRM/ledger (date, client, offer, amount, fees).
- Set aside ~25–30% of collected revenue for taxes (consult an accountant;
  this is an operational reminder, not advice).

## Hard money-safety rules (enforced by the sandbox too)
- The agent factory **never** charges anyone, moves money, or calls a payment
  API. It only *produces these documents and templates*. All real charges happen
  by you, in the PayPal dashboard, with your own credentials. (See SANDBOX.md.)
