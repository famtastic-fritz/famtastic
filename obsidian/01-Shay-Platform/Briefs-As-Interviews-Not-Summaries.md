# Briefs as interviews, not summaries

Date: 2026-06-15
Status: active doctrine

## Observation
- Fritz clarified that briefs should be interactive interviews.
- The purpose is to capture structured data that can formalize plans.
- Fritz challenged overuse of compact built-in memory because that layer is injected broadly and should stay lean.
- Fritz asked for an investigation into what currently nudges Shay to write there.

## Interpretation
- Durable evolving plan context should live in Obsidian/private artifacts with retrieval pointers.
- Compact built-in memory should only hold tiny standing facts that genuinely need every-turn injection.

## Proposed operating rule
1. Use brief/interview prompts to surface missing signal.
2. Capture the answers into a durable artifact.
3. Convert the artifact into a plan/workstream/issue.
4. Keep only the smallest stable preference/rule in built-in memory, if it truly deserves injection.
