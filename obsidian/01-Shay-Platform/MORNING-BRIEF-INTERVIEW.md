# Morning Brief Interview

Purpose:
This is the interactive briefing process. It is not a reminder dump.
It is how Shay gathers current truth, reduces pressure, updates plans, and produces a real brief.

## Trigger phrases
A fresh session should use this flow when Fritz says things like:
- start my morning briefing
- run my briefing
- let’s do the brief
- show me my latest briefing
- show me the plans we have

## Principles
- human state first
- pressure reduction over bureaucracy
- gather truth before reprioritizing
- do not force every answer into task language too early
- capture obligations, ideas, fears, deadlines, and blockers separately
- convert only what is actionable into plan/task updates

## Phase 1 — Human state
Ask first:
1. How’s your head this morning?
2. What feels heaviest right now?
3. Did anything happen since yesterday that changes the map?

If Fritz is overloaded, keep this phase short and move on.

## Phase 2 — Fire check
Ask:
1. Any money fire today?
2. Any deadline fire today?
3. Any relationship/family fire today?
4. Any work/client fire today?
5. Anything that feels spiritually or emotionally loud today?

Capture each as one of:
- observation
- obligation
- blocker
- emotional-weight signal
- decision needed

## Phase 3 — Plan-state check
Use ACTIVE-PLAN-REGISTRY.md as the default plan source.
For each active plan, check only what matters:
- did state change?
- what moved?
- what blocked?
- what now needs Fritz?
- what can move without Fritz?

Default active plans to review:
- NIBS Income Stability
- Cash Flow Danger Map
- Fast Revenue Lane Selection
- Cruise Go/No-Go Decision
- Lennox Scholarship and Recommendation Path
- Alec Job Path
- Household / Relationship Communication Relief
- Shay Capability Clarity
- Morning Briefing System
- MBSH Site Communication and Quality Decision

## Phase 4 — New-item intake
Ask:
1. Are there new items that need to enter the map today?
2. Is this a project, a task, a blocker, a promise, or just something we should not lose?
3. Does it belong to Income, Fritz, Research, Metaphysical, or Shay/Platform?

Routing rule:
- if bounded outcome -> attach/create project
- if executable -> create/update task
- if waiting -> queue
- if parallel research/build benefits -> swarm candidate
- if judgment-sensitive -> human-owned
- if missing dependency -> blocked
- if important but not for today -> parked / seed bank

## Phase 5 — Decision squeeze
Ask only the smallest set of questions needed to unlock movement:
- What one decision would reduce the most pressure today?
- What one task would create the most relief today?
- What one thing can Shay or another worker handle without you today?

## Phase 6 — Brief synthesis
Produce a briefing with these sections:
- Headspace summary
- Today’s fires
- Active plans and states
- Today’s direct actions
- Decisions needed from Fritz
- Queue-ready items
- Swarm candidates
- Blocked items with exact blockers
- Emotional weight / don’t lose
- Seeds parked consciously

## Phase 7 — Writeback
Update or generate:
- LATEST-BRIEFING.md
- ACTIVE-PLAN-REGISTRY.md if plan states changed
- any new project/plan artifact if a new bounded initiative emerged

## Command behavior
If Fritz says "show me my latest briefing":
- return LATEST-BRIEFING.md
- if a briefing is in progress, say that clearly

If Fritz says "show me the plans we have":
- return active plans first
- offer parked/seed-bank list second

If Fritz says "continue my morning briefing":
- resume from the most recent unfinished phase if known
- otherwise restart at Phase 1 briefly, then continue
