## Resolved Questions
Q: Why use JWTs instead of sessions? 
A: JWTs give us stateless auth across multiple Django apps without server clustering. No more session partitioning or sticky sessions needed.

Q: How are the tokens validated again on each request?
A: They hit our new TokenAuthentication middleware view, which extracts token from header and re-verifies it against HS256 hash of logged-in user's ID before routing to controller.
