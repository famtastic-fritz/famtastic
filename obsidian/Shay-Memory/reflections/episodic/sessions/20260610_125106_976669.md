The following technical decisions were made in implementing token-based authentication:
1. The use of JSON Web Tokens (JWT) provides a stateless method of maintaining user sessions without the need for persistent server-side storage between requests.
2. Signing tokens with a secret key adds an additional layer of security beyond encoding and protects against tampering or forgery.
3. Verifying the signature and claims contained within the token on each request greatly minimizes the risk associated with replay attacks.

While switching to JWT's will greatly simplify our authentication flow, migrating from sessions stored server-side in MySQL maintained by GoDaddy cPanel to tokens verified client side does significantly impact security design considerations concerning session management. The refactored system must be thoroughly reviewed and tested to ensure no vulnerabilities or regressions are introduced during the migration.

In the interest of improving overall application maintainability and simplifying logic, other suggestions for restructuring this part of the codebase to further abstract out the auth handling into a new service file have been considered but not yet implemented as part of this project scope. However, integrating such refactoring after initial JWT transition is highly recommended to continue improving future maintenance and extensibility of this critical component.
