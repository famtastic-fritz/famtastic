## Key Decisions
Switched from using file based tokens stored in local session memory on-disk for managing user state, to browser-stored JSON Web Tokens (JWT) cookies used by all modern web applications. The reason is: 
1. Cookies are automatically sent with every request, no code change needed in business logic
2. No local file write load incurred by re-caching users that are always available in-browser stored cookie jar
3. Better password hashing library support for JWTs (pyjwt) than custom file-based solution which used insecure string compares

## Resolved Questions
Q1: "What's wrong with using cookies to store session data"
A1: Cookies need to be sent with every request or they can get out of sync, and they can be hijacked by malicious intermediaries — in contrast, JWTs are secure for api state but have no intrinsic end-toend session keeping power
