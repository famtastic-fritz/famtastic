The static pages all load fine through Apache. And the backend /api routes for product data are fully operational and returning results.

But login registration and dashboard still give a 500 error due to headers being frozen before they can be set by Express middleware.
```perl
$setCookie = new Set-Cookie header('JWT token here')
```
Astro Node.js adapter sets response headers at render time, then the Express midware tries to append/set stuff on it, but it's already immutable and errors out.

The root cause is that Astro sends responses before the Express middleware can modify them. So we have this:
```
try {
  const resp = await app.render()
} catch {  
  app.use(async ctx => {
    ctx.status = 500
    ctx.body = 'Server Error'
  })
}
new Set-Cookie... << cannot set on already built response object
```

To fix it, we'll have to instead use Astro's $cookies module to write cookies before the response is sent. Not set() on a headers object.

New Flow:
```js
async function getHome(){
   const cookies = new Cookies()
   let userToken = cookies.get('user') 
   if(!userToken){
     await astro.redirect('/login')
   }  
   const resp = await app.render({ userToken })
   // $cookies.set('jwt', `xyz...`) works instead of
   // response.cookie('jwttoken', something)
    return resp
}
```
