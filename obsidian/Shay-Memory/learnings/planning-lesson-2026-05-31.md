# Planning Lesson: Swarm whole-file rewrite truncates large files; use it only for small/new files

**Score:** 6/10

## What went wrong
- multi_file_code_job had local brains rewrite 4 CSS files; SettingsShell (356L) was truncated mid-rule at line 211 (orphan '.'), ChatTabsRow (308L) silently dropped its trailing rule
- typecheck:web PASSED on the truncated file — only the vite/postcss BUILD caught the broken CSS

## Root causes
- Local-brain (Gemini/Ollama) output length cap truncates whole-file regeneration of files beyond ~250 lines
- typecheck does not parse CSS the way the bundler does — CSS truncation is invisible to it

## Fixes applied / to apply
- Route whole-file swarm rewrites ONLY to small (<250L) or brand-new files; StatusBar (51L) and a new Providers stylesheet succeeded
- For larger files use surgical_patch (anchored) not full rewrite, or section-by-section synthesis
- ALWAYS run the real build (not just typecheck) as the gate after CSS edits; add brace-balance + line-count-delta sanity checks
- Trust your own eyes over the vision model text — it gave false 'cut off on left' positives where the real issue was cramped/unstyled
