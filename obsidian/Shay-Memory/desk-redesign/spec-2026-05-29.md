# Shay Desktop — Redesign Spec

Captured through a 7-section interview on 2026-05-29. Each section locked the layout + features + behavior. Cross-cutting goals: power-user customizability, ToS-safe (no Claude-Code-mode imitation), token-efficient by default, match Claude Desktop UX where it already works.

Related: [[shay-shay]] · [[Section-1-bottom-row]] · [[Section-7-left-sidebar]]

---

## Cross-cutting principles

- **Drag-to-reorder + add/remove** is the default for any horizontal control strip (bottom row slots, panel tabs, sidebar sections, capture affordances).
- **Match current Claude Desktop behavior** where it already works — only deviate when there's a deliberate reason.
- **Don't violate Anthropic's third-party-tool ToS** — keep Chat mode as the canonical mode; extend it with terminal capabilities rather than mimicking Claude Code mode.
- **Token-efficient defaults** for attachments and context indicators (no auto-actions on context; show, don't act).
- **Unified "interactive block" pattern** — same chrome (header with pop-out / send-to-right / collapse + capped-height body with show-more) for terminal output, system asks, "run this for me" prompts.

## Section 1 — Bottom row (input controls)

```
[Slot A: up to 3 pinned actions] [+] [🎙] [Mode ▾] [Ctx 32k/200k] [Model · Think · $]
                                ⇅ all slots drag-reorderable, add/remove ⇅
```

| Slot | Role | Locked decisions |
|---|---|---|
| **A** | Left action | Up to 3 pinned actions side-by-side. Default: Accept edits. Swap menu: Plan mode, /compact, /clear, restore checkpoint, switch brain, custom slash command, hide. |
| **B** | `+` add menu | Attach file · paste image · insert skill · run MCP tool · insert snippet · trigger routine · new chapter marker. |
| **C** | 🎙 mic | Push-to-talk · dictation toggle · one-shot send-on-pause · live transcription · voice mode w/ TTS reply. |
| **D** | Mode ▾ | Chat / Cowork / Code (existing) + future presets if needed. Hide-able. |
| **E** | Context indicator | **Standalone slot between D and the model pill.** Shows tokens used / total + %. **Display only — no auto-action.** |
| **F** | Model pill | Click model → switcher (Claude / Codex / Gemini / Ollama). Click thinking → Low/Med/High/Ultra. Live $/turn. Context % already in Slot E. |

**Reorderability:** full drag-reorder + add/remove of slots.

## Section 2 — Text editor entry row

### Inline triggers (all enabled)
- `/` slash commands
- `@` mentions (files, skills, agents, MCP tools)
- `!` inline shell preview
- ``` `` ``` code fence with auto syntax highlight + live markdown render

### Behaviors
- **Send key**: `Enter` sends, `Shift+Enter` newline.
- **Edit last**: BOTH — `↑` on empty input pulls most recent · hover any past message → "Edit" button forks from there.
- **History**: ALL sessions, fuzzy-searchable, with multiple search scopes (project / brain / mode filters available).
- Auto-resize · drag-drop / paste anywhere · OS-native spell check.

## Section 3 — Media row (above input)

**Empty state: hidden completely** (max chat real estate). Capture affordances live in Slot B (`+` menu) when row is empty.

When attached:

```
[📷 thumb ✎ ✕ ⓘ] [📄 thumb ✎ ✕ ⓘ] [🎙 0:14 ✎ ✕ ⓘ]  drag to reorder
```

### Per-chip actions
- ✎ Annotate (lightweight in-Desk: arrow, box, blur, crop, trim, highlight)
- ⓘ Inspect — see what Shay will actually send (OCR, transcript, summary, raw)
- ✕ Remove
- ⤴ Reuse from past attachments

### Capture tools (configurable — user toggles which appear)
- 📸 Screenshot (region / window / full)
- 📹 Screen record (short clip + voice)
- 🎥 Webcam (photo + short video)
- 🎙 Voice memo (attached as audio, separate from Slot C dictation)

### Smart-handling defaults: token-efficient
Image → OCR + brief describe · PDF → first page + TOC + chunk summaries · Audio → transcribe · Video → key frames + transcript · URL → main content fetch · Code → relevant excerpt with tree context. Per-chip ⓘ menu to override.

## Section 4 — Chat window

### Long-message handling
- **Auto-collapse over ~50 lines with `[show more]`.**
- Smart-collapse by block type (prose full · tool calls collapsed · code over 30 lines collapsed · thinking hidden).

### Block types and behavior

| Block | Default |
|---|---|
| Tool call (MCP, file edit, web fetch) | **Collapsed one-liner**, click to expand input + output |
| Code | Syntax highlight · copy · run · save · open in editor · diff with current |
| File diff | Inline OR side-by-side · accept/reject per hunk |
| Thinking | Hidden by default; global + per-message toggle |
| Terminal output | **Capped-height with show-more**, header row with pop-out · send-to-right · collapse · copy |
| Image / audio / video | Inline with click-to-fullscreen, scrub-to-quote on audio/video |

### "Interactive block" reusable pattern
The terminal-output block is the *same component* used for:
- Shay system asking the user something
- "Run this in your terminal" prompts
- Any case where Shay needs user input mid-response

Always has the header chrome (pop-out / send-to-right / collapse) and a capped body with show-more.

### Window behaviors
- Sticky-to-bottom unless user scrolls up (`↓ new` pill appears)
- Chapter gutter markers · `⌘F` in-chat fuzzy search
- Quote-to-reply (select → "Reply with this quoted")
- Branch / fork from any past message (right-click → "Branch from here")
- Save / bookmark messages
- Copy as: markdown / plain text / JSON / shareable link

### Pop-out: **manual only** (right-click → "Pop out"). No auto-pop.

## Section 5 — Top bar (session switching)

```
◀ ▶  🔍   ⌂ project ▾ /  Session-name ▾   • Opus   ⏱ 3m  📌  ⤢ split  ⋮
```

### Switcher: BOTH dropdown + ⌘K palette
- Click session name → visual dropdown (recent · pinned · search · "New" · folders/tags)
- `⌘K` → fuzzy text palette across sessions / projects / chapters

### Organization: FLAT, most-recent first
Pinned float to top. No mandatory folders. Per Section 7, custom sections in sidebar can re-cluster.

### Multi-session: SPLIT-PANE + TABS
Tabs across top of chat area (`⌘⇧]` / `⌘⇧[` to cycle); right-click any tab → "Open in split". Side-by-side comparison / parallel work.

### Project chip behavior
**Switching project starts a NEW session bound to that project.** Current session stays bound to its original project (no scope confusion).

### Picker row info
Title (inline editable) · project chip · brain icon · last activity · msg count · chapter count · status (🟢 running · ⚪ idle · 📌 pinned · 🗄 archived).

```
┌───────────────┐  ┌─────────────────────────┐
│               │  │  Preview ▾  ⤢ 📌 ✕    │
│   chat        │  ├─────────────────────────┤
│               │  │   active panel content   │
│               │  ├─────────────────────────┤
│               │  │  🔴 3 running · ⚠ 1 input│
│               │  │  ✓ 12 done  ↓ details   │
└───────────────┘  └─────────────────────────┘
```

### Available panels
Preview · Diff · Terminal · Files · Background tasks · Plan · `+ Custom` (pin any MCP output, saved query, vault-search result).

### Panel switcher: **TABS + vertical-split**
Reorderable, pinnable, closable tabs across top. **Plus ability to split the panel vertically** to show two panels stacked (e.g. Preview top + Terminal bottom). Tabs above each pane.

### Auto-switching: **match current Claude Desktop UX**
Whatever Claude Desktop does today (auto-focus Terminal on cmd run, Diff on staged edits, etc.) — preserve that behavior. No surprises.

### Task tray: **default to current Claude Desktop pattern**
Unless explicitly overridden, mirror Claude Desktop's current task/notification surface.

### Notifications: configurable per category + DND windows
- Per-category rules (input-needed · task-done · task-failed · message-arrived)
- Scheduled quiet hours
- Sound opt-in per category
- Per-task override ("ping me when this finishes")
- OS-level (desktop banner + dock badge) vs in-app pulse

## Section 7 — Left sidebar

```
💬 Chat   📋 Cowork   </> Code        ← mode tabs (collapsible to icons)
─────────────────────────
+ New session
⚡ Routines
✏ Customize
⋯ More
─────────────────────────
📌 Pinned
  • Quick brainstorm
─────────────────────────
🕐 Recents
  • Session A
  • Session B
─────────────────────────
👤 Fritz · Max  ▾             ← profile / account
```

### Mode tabs: keep Chat / Cowork / Code where they make sense for our needs
**No new top-level modes invented.** Match current Claude Desktop concepts. **Important addition: under Chat, add terminal capabilities** — gets the power of Cowork/Code's shell access while keeping Chat's auth/ToS framing (avoids violating Anthropic's third-party-tool ToS by not imitating Claude Code mode).

### Sidebar collapse: TWO-STAGE
- Click chevron → icon-only column (hover reveals labels)
- Double-click → fully hidden (hover left edge to peek)

### Section list elements
| Element | Behavior |
|---|---|
| **+ New session** | Primary CTA. Picker for project · brain · mode, OR start blank in current mode. |
| **⚡ Routines** | Saved auto-runs / cron jobs / scheduled prompts. Active · paused · history. |
| **✏ Customize** | Theme · density · shortcuts · which bottom-row slots appear · which capture tools appear · panel layout. |
| **⋯ More** | Plugins · MCP Registry · Skills browser · Logs · Help · About. |
| **📌 Pinned** | Drag-reorderable starred sessions. |
| **🕐 Recents** | Flat-by-recency list. Right-click: rename / archive / delete / pin. |

### Bottom user/profile menu (ALL OF THESE)
- Account + Plan & billing
- Brains & MCP servers status (live green/red dots, quick-toggle each)
- Theme + appearance quick-switch
- Lock + Sign out

### Custom sections: YES — fully customizable
Drag to reorder. Add/remove. Define your own that surface anything (recent vault notes via [[basic-memory]] + [[vault-search]], project shortcuts, MCP query results, MBSH stuff, etc.).
