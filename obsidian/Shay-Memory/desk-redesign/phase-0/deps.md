## Conflicts / deviations from the build plan

- **lucide-react** — Build plan requested `^0.500`. The repo already had
  `lucide-react ^1.7.0` declared and installed. Upstream `lucide-react`
  cut a 1.0 line in May 2026 (the `0.5xx.x` series ended at 0.577.0 before
  the 1.0.0 release). `^1.7.0` is a superset of the 0.5xx API for icon
  imports (named exports `<Icon name="..."/>` and per-icon imports still
  work the same way), and `react@19` is supported. **Kept `^1.7.0`** to
  avoid a no-op downgrade; the Phase 0 icon registry work can target the
  1.x API directly.
- **tinykeys** — Plan asked for `^3`. Latest is 4.0.0 but the plan
  explicitly pinned `^3`, so pinned `^3.0.0` (resolved to 3.1.0, the
  highest 3.x). 4.0.0 has a breaking change in the `tinykeys()` callback
  signature; staying on 3.x matches the plan.
- **@dnd-kit/sortable** — Plan asked for `^9`. Latest is 10.0.0 but
  10.x requires `@dnd-kit/core@^7` (not yet released as stable at install
  time). Pinned `^9.0.0` (resolved 9.0.0), matching `@dnd-kit/core@^6`.
  Matches the plan exactly.
- **react-resizable-panels** — Plan asked for `^2`. Latest is 4.x but
  4.x ships a new imperative API the rest of the build plan does not
  reference. Pinned `^2.1.9` (resolved 2.1.9) per the plan.
