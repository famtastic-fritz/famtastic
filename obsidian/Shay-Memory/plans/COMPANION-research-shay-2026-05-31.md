## Vision (clone+exceed)
The Shay phone companion aims to first achieve full capability parity with leading AI chat applications like Claude, ChatGPT, and Codex, providing seamless natural language interaction for queries, code generation, and content creation, directly building on the existing `shay-phone` architecture. Its transformative advantage, however, will be realized by exceeding these incumbents through deep integration with Shay's desktop agent-OS, functioning as a true remote control for real-world project execution, encompassing build dispatch, real-time mid-run workflow approvals, access to persistent vault memory, comprehensive desktop agent control, agent-OS job monitoring via heartbeats, robust self-update mechanisms, and a brain-agnostic architecture, thereby shifting from a passive chatbot experience to active, operational AI management.

## Workflow Ties
The Shay companion app directly integrates with and amplifies Shay's core desktop workflows, transforming the phone into a comprehensive remote control. Voice and text inputs (via `/api/chat`, `/api/ask`) initiate `build dispatch` tasks, query `vault memory` for project context, and issue `desktop/agent-OS control` commands, allowing users to articulate complex instructions or retrieve specific information instantly. `Mid-run approvals` are facilitated through push notifications and interactive elements on job status screens (powered by `/api/job/*` endpoints), enabling real-time gating of Shay's execution pipeline. Critical `heartbeats` and status updates, including `self-update` notifications, leverage the monitoring spine (`notify.py`) to ensure the user is always informed of Shay's progress and state. This `brain-agnostic` architecture ensures that all interactions, regardless of the underlying AI model, contribute to and benefit from Shay's persistent operational memory and real-world execution capabilities. Every interaction, from a voice command to a text approval, is funneled through Shay's robust backend to manifest as concrete actions on the desktop or as entries in the persistent vault.

## Top-10 Differentiators

Shay's companion app decisively beats generic chatbot applications like Claude, ChatGPT, and Codex by leveraging her unique desktop agent capabilities and workflows: (1) **Real-World Project Execution (Build Pipeline):** Remotely dispatches actual `build` pipeline runs, utilizing Shay's `dispatch` and `job/*` APIs for complex desktop tasks (e.g., code compilation, deployment, data processing) far beyond theoretical chat responses. (2) **Mid-Run Approvals & Intervention:** Provides real-time `mid-run approvals` for ongoing desktop processes, enabling critical decision-making and course correction directly from the phone app. (3) **Persistent Vault Memory & Context:** Accesses and leverages Shay's `vault memory`, grounding conversations and commands in a deep, evolving understanding of the user's projects, preferences, and prior interactions for hyper-contextual actions. (4) **Desktop/Agent-OS Control & Monitoring:** Functions as a true remote control for the `agent-OS`, providing visibility and command over desktop applications, file systems, and background processes via `/api/arc` and `/api/job/*`, seamlessly bridging the phone-desktop gap. (5) **Brain-Agnostic Intelligence Layer:** Operates as a `brain-agnostic` interface, allowing the user to select or switch underlying large language models (LLMs) and tools on the desktop side, ensuring flexibility and avoiding vendor lock-in. (6) **Self-Updating Agent & Infrastructure:** Enables remote initiation and monitoring of Shay's `self-update` mechanisms, ensuring the desktop agent and its associated toolchain are always current. (7) **Real-Time Heartbeats & System Health:** Displays `heartbeats` and operational status of the desktop agent via `notify.py` integration, offering transparent system health and responsiveness metrics crucial for mission-critical automation. (8) **Automated Workflow Dispatch & Triggering:** Facilitates triggering of complex, multi-step `dispatch` workflows and chained commands, transforming phone input into orchestrated sequences of desktop actions. (9) **True Personalization via Local Data:** Leverages local data, configurations, and environment variables residing on the user's desktop for deeply personalized interactions and actions, creating a private, highly customized assistant. (10) **Proactive Notifications & Signoffs:** Receives proactive notifications and `signoffs` from Shay regarding task completions, errors, or required user input (via `notify.py`), turning the phone into an active, intelligent command center.

## Per-Unit Recipe + Gates
Each build unit follows a consistent recipe and gating process: First, update relevant Python server-side code and dependencies (`shay-phone/server.py`, new modules) including API endpoint extensions; second, update associated PWA client-side components (`web/index.html`, `arc.html`, `sw.js`, new JS modules) to consume new API functionality and enhance UI/UX. A unit is gated for completion only after successfully passing `py_compile` for all Python changes, then `server-boot /api/ping` verifies server liveness and API availability, and finally `node --check` validates client-side JavaScript syntax and dependencies.

## Brain-Agnostic + Self-Update
The `shay-phone/server.py` acts as a brain-agnostic abstraction layer, exposing a consistent API (e.g., `/api/chat`, `/api/dispatch`) to the PWA regardless of Shay's underlying large language model or intelligence backend, thereby preventing vendor lock-in and allowing seamless integration of future AI systems. All new capabilities will extend this existing API, ensuring the PWA remains decoupled from specific AI implementations. For self-update, the companion app leverages Shay's existing `build dispatch` workflow: server-side updates are managed by Shay dispatching jobs to pull new `server.py` code and restart the service, gated by `py_compile + server-boot /api/ping`, while the PWA's `sw.js` actively monitors a `/api/version` endpoint or asset hashes, prompting it to fetch, cache, and activate new `web/index.html` and associated assets verified via `node --check`, ensuring the client is always current with Shay's latest builds.

## Risks
Integrating Shay's full desktop capabilities (build dispatch, mid-run approvals, vault memory, agent-OS control) into the phone app introduces significant complexity and potential latency issues, especially for real-time voice and desktop interaction over variable mobile network conditions. Expanding the API to expose these core functionalities necessitates rigorous security hardening to prevent unauthorized `dispatch` or `job/*` manipulation, increasing the attack surface. Relying primarily on a PWA for "exceed" features might encounter inherent browser limitations, potentially failing to meet user expectations established by native incumbent apps, while tight coupling to Shay's evolving desktop agent functionalities risks versioning conflicts and introduces a single point of failure.
