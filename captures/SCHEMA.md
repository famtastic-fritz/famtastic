# Capture Packet Schema — v0.2.0

A capture packet is the normalized output of any capture adapter (Claude Code, Cowork, Codex, manual paste). All adapters emit this shape into `captures/inbox/<id>.json`. The promoter reads from `captures/review/<id>.proposal.json` (intermediate proposal generated by `fam-hub memory review`).

# Netlify cannot link a project to a Git repo via API

The Netlify "connect to git" flow requires the vendor UI. The Netlify REST API
does not expose project ↔ Git repo linking. `netlify api updateSite` can rename
a project, change build settings, etc., but cannot establish or change the
linked GitHub repo.

## Workaround

Use the assisted-manual capability pattern: open the deep-link URL
`https://app.netlify.com/sites/<site>/settings/deploys` and poll the API to
detect when the repo link is established.

## Evidence

- Session 2026-05-05 — `netlify api updateSite` returned `repo: null` after rename
- `platform/capabilities/deploy/connect-netlify.sh` documents the limitation in comments

## Versioning

- v0.1 — implicit; what `capture-insights.js` produces today (5 hardcoded buckets)
- **v0.2** — this doc; adds tags, confidence, structured source, types, lifecycle

Adapters emit `schema_version` so the promoter can route legacy v0.1 packets through a migration path.
