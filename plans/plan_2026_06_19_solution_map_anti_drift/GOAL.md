# Goal — Shay solution-map anti-drift

Primary outcome:
Create a canonical connection surface Shay can query before doing fresh research so prior evidence, candidate repos, repeated failures, and unresolved proof gaps are reattached to the current problem.

What this goal means in practice:
- A recurring problem should resolve to a canonical bucket, not a fresh pile of prose.
- Each bucket should show source evidence, repeated mentions, candidate solutions, proof status, and closeout state.
- New research should be additive only after the prior bucket context has been checked.

Anti-drift system:
1. Observations and interpretations stay separated in every audit artifact.
2. Problems are normalized into canonical buckets with aliases.
3. Capability missing vs capability unproven are separate states.
4. Promotions/closeouts are explicit ledger states, not implied by notes.
5. Repo discovery is not enough; every promising repo needs a verdict tied to a real FAMtastic problem.
6. Truth docs must name the connection surface so later sessions know where to look first.
