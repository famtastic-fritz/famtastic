# Phase 2: Visual Workflows & Brand Systems

Status: current plan of record after Phase 1 Waves 1–7.

## Purpose

Phase 2 turns the Phase 1 foundation into useful visual workflows. It should not start with an abstract dashboard. It starts with real value: FAMtastic's logo/brand system and a usable Site Studio build/edit/enhance workflow.

## Current order

1. Media Studio through FAMtastic logo/brand creation
2. Site Studio useful build/edit/enhance workflow
3. Component Studio from repeated Site/Media needs
4. Mission Control visual orchestration
5. Data Center / Research Center / Second Brain visual UI
6. Shay Desk Office tab integration

## Non-negotiable platform contract

Every platform-level component inherits this loop:

- research first
- spec-shaped execution
- reuse before generate
- route specialized needs to the owning studio
- return structured results to the caller
- record proof, costs, citations, prompts, outputs, and where-used metadata
- run post-evaluation after meaningful runs/builds/jobs
- identify opportunities for skills, processes, components, media recipes, routing rules, and workflow improvements
- save reusable artifacts back into the owning studio/library
- send learning/proposal records to Data Center for review or promotion

## Phase 2A: Media Studio via FAMtastic logo/brand creation

Goal: create the real FAMtastic logo/brand system through Media Studio, not through copy/paste/manual one-offs.

Starting inputs already present:

- `brand/FAMTASTIC-BRAND-MARK.md` — current brand mark manifesto and production rules.
- `remotion/` — current FAMtastic logo motion composition source and recipes.
- `docs/archive/pre-shay-shay/local-artifact-manifests/epic-mclean-muapi-logo-outputs.md` — manifest for local MuAPI logo/media WIPs preserved outside the repo.

Workflow requirements:

- brand brief
- research/taste references
- visual direction and constraints
- prompt lab / prompt versioning
- provider/model selection
- generation queue
- variant grid
- critique/scoring
- iteration history
- approved asset
- export package
- DESIGN.md / brand tokens
- where-used tracking
- post-evaluation opportunity capture

## Phase 2B: Site Studio useful build/edit/enhance workflow

Goal: support a request like “build me a website for coconut sales in the Bahamas” through the intended workflow.

Workflow requirements:

- capture brief
- research market/audience/competitors
- shape spec
- extract media needs
- extract component needs
- search/reuse before generating
- build site
- preview
- edit specific sections
- enhance with media/assets/components
- publish/update path
- measure/learn/post-evaluate

## Phase 2C: Component Studio from real reuse pressure

Component Studio should be shaped by repeated needs discovered during real Site/Media workflows. Likely candidates include heroes, slideshows, galleries, pricing cards, testimonial blocks, CTA strips, comparison tables, lead forms, trust badges, media cards, and animated affordances.
