# Site Planning Docs

Cross-site planning material lives here. Per-site working files (assets,
build artifacts, runtime configs) stay in `sites/<tag>/` as the architecture
intends. Strategic planning that future factory sites will reference — design
language, character sheets, sponsor architecture, tone guides — lives here
in the hub repo so the signal isn't locked inside an archived per-site repo.

## Filename convention

Site planning docs use uppercase canonical names (e.g.
`HI-TIDE-HARRY-CHARACTER-SHEET.md`, `MBSH-REUNION-PLANNING-CONSOLIDATED.md`)
to match their original artifact filenames and remain greppable across the
project.
