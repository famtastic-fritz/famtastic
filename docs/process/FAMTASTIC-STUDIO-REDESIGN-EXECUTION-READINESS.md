## Recommended First Implementation Slice

After review, the first implementation slice should be the lowest-risk slice that proves the product model without launching a downstream site:

1. Static/readonly Studio navigation and screen shell map.
2. Mission Control attention model with mock data.
3. Shay visible home placeholder linked to Bubble/Desk concepts.
4. Capability Truth readonly status model with proof links.
5. Run Ledger/Trace readonly separation.

This is not an approval to code. It is the recommended slice once the spec map is reviewed.
