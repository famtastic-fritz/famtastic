Discovery — applied below as five screens

Five surfaces, one system.

Each screen below is a different domain from the foundation rules, all using the same Night theme tokens, the same vocabulary, and the same rules. Discovery answers to your specific questions are embedded inline. Scroll through to see how the same design language renders across very different jobs.

What is "creating the FAMtastic logo"?

It's a specific case of the Media · Generate flow with branded story tier. (See Screen 4.) The flow is:

  • Pick the goal (logo, hero, character, voice, etc.)
  • Capability manifest routes to candidate models — for a logo, primary is Claude SVG + OpenAI Responses (validated)
  • Models generate in parallel, results compare side-by-side
  • You pick the winner — winner promotes to brand kit, runners-up archive
  • Permission gate fires before any external API spend over a threshold

How do you pick different models?

You don't pick models — you pick a goal. The capability manifest (`studio-capabilities.json`) routes goals to validated providers. Manual override is available for experiments, but the default UI surfaces the capability tier (validated / fallback / partial / broken) so you know what you're picking before you click.

How do comparisons work?

Multi-model generation is the default for any branded asset. 2–4 cells in a grid, same prompt, different routes. Cost shown per cell. Click a winner — it promotes. Losers stay in history for one session, then archive. (Screen 4 shows this.)

Permission gates — when do they fire?

Three triggers: spend over threshold, write to brand kit (canonical), or any provider in `unverified` tier. Gate appears as a contextual approval bar inside the workspace, never as a modal. Approve once = scoped to this generation. (Bottom of Screen 4.)

Components vs Media — what's the line?

Components are AI-generated reusable code/HTML modules (hero sections, cards, badges) — versioned, tested, promoted to library after proof. Media are static assets (images, video, audio, brand kit) — generated, versioned, owned by a site or by the brand. Both have library + generate + detail flows. Different domains because acceptance criteria differ.

Where do jobs live?

The job queue isn't a left-nav domain. It's a panel inside Admin (worker queue) and surfaces contextually wherever a job is dispatched (e.g., "generation queue" in Media, "build queue" in Sites). The same underlying ledger, three contextual views.
