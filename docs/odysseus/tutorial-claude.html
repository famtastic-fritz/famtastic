Odysseus

Your private, self-hosted AI workspace — the ChatGPT/Claude UI, but running on your own Mac, on your own data. Here's why it earns a spot next to FAMtastic Studio, and how to actually get value from it.

Tutorial 1 of 2 · authored by Claude Code · the build-companion view · 2026-06-02

1 The one-sentence why

Everything FAMtastic leans on the cloud for — chat, research, agent runs — now has a local mirror you fully own. When an API is down, rate-limited, or you're working on something that shouldn't leave the box, Odysseus covers you. Think of it as a private second brain that never phones home.

3 Get it running in 90 seconds

1

Install

bash ~/famtastic/scripts/odysseus/install-odysseus.sh --start — clones, seeds a private .env, and launches on Apple Silicon (GPU-capable).

2

Open & log in

Visit http://127.0.0.1:7860. Grab the temp admin password printed in your terminal, log in, and change it in Settings.

3

Add a model

Settings → providers. Point at Ollama/OpenRouter/OpenAI, or open Cookbook, let it scan your Mac, and one-click a model that fits your VRAM.

4

Drive it via the agent

From Claude Code, the odysseus subagent knows how to install, start, health-check, and route work here. Just ask.

4 When to reach for Odysseus vs. stay in the cloud

The honest decision rule — local isn't always the answer.

Go local →

Sensitive data you don't want leaving the box · bulk/low-stakes generation · offline or API-down moments · running models at zero marginal cost · email/calendar/notes ops · blind model comparisons.

Stay cloud →

Top-tier reasoning (latest Opus) · very large context windows · the FAMtastic site-build pipeline (Studio is tuned for it) · anything where quality > privacy/cost.

5 Five power moves

1. Make it your private research desk. Deep Research runs the whole gather→synthesize loop locally and hands you a visual, cited report — no tokens spent, nothing logged off-box.
2. Run a model bake-off before you commit. Use Compare blind-test to pick the right local model for a job, then serve the winner from Cookbook.
3. Bridge it to the brain. Odysseus memory exports cleanly; pull it into obsidian/ and push your skills the other way so both brains learn once.
4. Turn your inbox into an agent surface. Email triage + Notes/Tasks means the agent can summarize, draft, and act on reminders — great for the client-ops/acquisition side.
5. Keep it on a leash. The Agent has shell access by design. Scope its tools like Claude Code's, keep it on loopback, rotate that first password.
Safety in one line: loopback-only by default, auth always on, never expose port 7860 to the public internet — LAN/Tailscale only, and rotate the first-run admin password immediately.
