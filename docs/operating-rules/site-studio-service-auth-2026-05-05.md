# Site Studio Service Auth — 2026-05-05

## Decision

Provider authentication belongs to Site Studio/platform, not to generated sites.
Sites consume Studio-provisioned services through generated config and vault
references.

This corrects the MBSH launch-auth confusion: MBSH is a product generated by
Site Studio. It is not the owner of Netlify, Resend, DNS, cPanel, SSH, or DB
provider accounts.

## Commands

```bash
fam-hub platform bootstrap-services
fam-hub platform bootstrap-services --check
fam-hub platform provision-site mbsh-reunion-v2 --check --proof
```

`bootstrap-services` checks provider auth and migrates discoverable local
secrets into the platform vault without printing values. `provision-site`
verifies that a site consumes Studio-owned services and can emit a proof packet.

## Vault Ownership

Studio-level provider secrets use Studio IDs:

- `studio.resend.api_key`
- `studio.cpanel.api_token`
- `studio.netlify.auth_token`
- `studio.db.admin_password`

For the current GoDaddy-hosted stack, `studio.cpanel.api_token` is the primary
hosting/DB/DNS control credential. `studio.godaddy.api_key` and
`studio.godaddy.api_secret` are optional registrar/direct-DNS fallback refs, not
the required default path.

Generated site credentials use site references:

- `sites/<tag>/db_password.<env>`
- `sites/<tag>/db_ref.<env>`
- `sites/<tag>/netlify_site_id`

Lower-level platform helpers must prefer these Studio-owned IDs. Legacy IDs
such as `resend.api_key`, `godaddy.api_key`, or `netlify.auth_token` may remain
fallbacks for old local setups, but new provider wiring uses `studio.*`.

## Current MBSH Result

The 2026-05-05 bootstrap migrated the existing local Resend API key, cPanel API
token, MBSH production DB password, and MBSH production DB reference into the
vault. Resend verified with a successful `/domains` probe.

MBSH provision proof is at:

- `proofs/studio-service-auth-mbsh-reunion-v2-2026-05-05.json`

Remaining blockers:

- `config/site-config.json` still has `API_BASE_URL: null` until Studio
  generates the production API origin after backend provisioning.
- DNS/addon-domain automation still needs cPanel UAPI/MCP wrapper coverage;
  until then, DNS may remain a necessary cPanel/GoDaddy UI step.
- SSH to `nineoo@FAMTASTICINC.COM` is blocked by host-key verification.

## Rule

Do not create future site-specific provider ownership plans for Netlify,
Resend, DNS, cPanel, SSH, or database admin credentials. Do not ask for GoDaddy
developer API keys when cPanel is the working control plane. Add capability gaps
to the Studio/platform layer, then let generated sites consume those
capabilities.
